# notes/ — the decision log

**Status: record, not canon.** These files capture the design sessions — the clarifying
questions, the user's locked answers, and the audits that shaped the v2 docs. They exist so
that nobody (human or agent) re-litigates a settled question or quietly re-decides it
differently. **On any conflict with `docs/`, the docs win** — the docs are the source of
truth for *what* the design is; these notes record *why* and *that it was decided*.

- `00-state-assessment.md` — triage of the 2026-06-09 reboot (what was deleted, what the brief asked for).
- `01-decisions.md` — the locked answers from clarifying rounds 1–3 (structural / fighting system / party & meta).
- `02-final-answers.md` — round 4 free-form answers + the fog re-theme naming.
- `03-library-policy.md` — the external-libraries directive and the tech-plan audit against it.
- `scratch/` — gitignored; genuinely throwaway working space.

New entries: number them in sequence, date them, and keep them append-mostly — a decision
log you rewrite is no longer a log. If a note here is superseded, say so in place rather
than deleting it.
