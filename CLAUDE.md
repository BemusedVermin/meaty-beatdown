# Repo orientation

This repository contains the language-neutral design plus a **Rust implementation** of the engine.
Work within whichever subfolder is relevant, and read that folder's own instructions.

- **`docs/`** — the design (source of truth: `frame_rpg_spec.md`) + the mechanics docs. Language-neutral.
- **`golden/`** — the cross-language golden vectors (the behavioral contract). Language-neutral; shared.
  These were generated by a now-removed TypeScript reference implementation (recoverable from git
  history); the Rust engine must replay them byte-identically.
- **`rust/`** — a from-scratch rebuild the user is implementing themselves (see `rust/DESIGN.md` for
  the model and `rust/README.md` for how to build/run). A two-crate workspace on **edition 2024**
  (needs Rust ≥ 1.85):
    - **`engine`** — the Bevy-free deterministic core (deps: `bevy_math`, `hexx`, `fastrand`,
      `fixed`). Modules: `fighting` (the turn-based combat sim), `exploration` (the seeded ocean
      hexcrawl), and `content` (the L4 RPG layer — the `Build → Fighter` compiler; stats/skills/
      equipment compile into the **moves** that are the unit of combat).
    - **`app`** — the Bevy (`0.18`) game shell that *drives* the engine: the FSM axes (`state/`), the
      `combat` + `exploration` drivers, `render` (2D presentation), and `debuglog` (a file trace).
  It is **playable** today — `cargo run -p app` boots into the overworld; sail into an encounter and
  a fight runs end-to-end. The dependency arrow runs one way (`app → engine`); the engine never
  depends on Bevy. Default to scaffolding, reviewing, and answering questions here — do **not** write
  the engine logic unless explicitly asked. Run `cargo` commands from inside `rust/`.
